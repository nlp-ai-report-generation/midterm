# Project Orchestration

## 사용 시점

- 무엇을 먼저 만들지 정해야 할 때
- 주차별 마일스톤이나 제출물 중심으로 작업을 정리할 때
- 구현, 문서, 발표 준비를 하나의 계획으로 묶어야 할 때

## 필요한 입력

- 루트 `README.md`
- `docs/decision-points.md`
- `memory/current-state.md`
- `memory/decisions.md`

## 작업 절차

1. 현재 목표와 남은 제출물을 파악한다.
2. 작업을 중간평가, 최종평가, 데모 준비 기준으로 다시 묶는다.
3. 논의가 필요한 항목과 이미 결정된 항목을 분리한다.
4. 다음 액션을 1주 단위 또는 세션 단위로 재정렬한다.

## 기대 산출물

- 실행 가능한 우선순위 목록
- 짧은 구현 계획
- 평가 기준과 연결된 작업 정리

## 가드레일

- 과제 범위를 벗어나는 기능 확장은 기본값으로 제안하지 않는다.
- 흥미로운 기술보다 평가 임팩트가 큰 작업을 우선한다.
- 기존 결정과 충돌하면 먼저 충돌 지점을 드러낸다.
