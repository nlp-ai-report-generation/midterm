# Demo And Docs

## 사용 시점

- README나 과제 설명 문서를 보강할 때
- 발표 자료, 시연 흐름, 보고서 문안을 정리할 때
- 평가자 관점에서 메시지 전달력을 높여야 할 때

## 필요한 입력

- 루트 `README.md`
- 과제 폴더 `README.md`
- `docs/decision-points.md`
- 최신 메모리 문서

## 작업 절차

1. 이 문서가 누구를 위한 문서인지 먼저 정한다.
2. 전달해야 할 핵심 메시지 3개 이내로 압축한다.
3. 기술 설명보다 문제 해결 가치와 결과를 앞세운다.
4. 평가 기준과 연결되는 문장을 남긴다.

## 기대 산출물

- 읽기 쉬운 프로젝트 문서
- 발표/시연용 구조 정리
- 평가 대응용 설명 문안

## 가드레일

- 문서를 과도하게 장황하게 만들지 않는다.
- 내부 메모와 외부 공유용 문서를 섞지 않는다.
- 주장만 있고 예시나 산출물이 없는 문장은 줄인다.
