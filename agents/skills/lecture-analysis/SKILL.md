# Lecture Analysis

## 사용 시점

- 강의 스크립트를 체크리스트 기준으로 읽어야 할 때
- 분석 항목, 점수 체계, 휴리스틱을 정의할 때
- LLM 분석 결과의 근거와 한계를 정리할 때

## 필요한 입력

- 과제 폴더 `README.md`
- `강의 품질 기준.md`
- 강의 스크립트
- 강의 메타데이터
- `memory/domain-model.md`

## 작업 절차

1. 분석 대상 강의와 메타데이터를 확인한다.
2. 체크리스트 5개 카테고리와 18개 항목 중 이번 작업 범위를 고른다.
3. 항목별로 규칙 기반 검출 가능성, LLM 판단 필요성, 불확실성을 나눈다.
4. 근거 문장, 해석, 개선 제안을 분리해서 정리한다.

## 기대 산출물

- 항목별 분석 메모
- 점수화 기준 초안
- 증거 기반 평가 문안

## 가드레일

- STT 오류를 실제 강의 품질 문제로 단정하지 않는다.
- 스크립트에 없는 맥락을 상상해 보완하지 않는다.
- 하나의 사례를 전체 강의 특성으로 일반화할 때는 근거 범위를 적는다.
