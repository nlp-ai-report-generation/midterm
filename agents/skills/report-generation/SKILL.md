# Report Generation

## 사용 시점

- 분석 결과를 강사나 평가자가 읽을 문서로 바꿀 때
- 리포트 템플릿, 요약 구조, 개선 제안 섹션을 설계할 때
- 샘플 리포트나 대시보드 문구를 작성할 때

## 필요한 입력

- 분석 결과 또는 프롬프트 출력
- `강의 품질 기준.md`
- `memory/current-state.md`
- 관련 결정 문서

## 작업 절차

1. 보고 대상이 강사인지 평가자인지 먼저 확인한다.
2. 핵심 요약, 주요 관찰, 근거, 개선 제안 순서로 구조를 잡는다.
3. 각 제안이 실제 행동으로 옮겨질 수 있는지 점검한다.
4. 전문 용어를 줄이고 중요한 메시지를 먼저 배치한다.

## 기대 산출물

- 강사용 리포트 초안
- 요약 카드 또는 대시보드 문구
- 샘플 분석 결과 문서

## 가드레일

- 추상적인 칭찬이나 비판만 남기지 않는다.
- 점수만 제시하고 설명을 생략하지 않는다.
- 개선 제안은 관찰된 문제와 직접 연결되어야 한다.
