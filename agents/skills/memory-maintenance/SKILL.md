# Memory Maintenance

## 사용 시점

- 의미 있는 계획, 결정, 구현, 문서 변경이 끝났을 때
- 다음 에이전트가 바로 이어받을 수 있게 상태를 남겨야 할 때
- 열린 질문과 확정된 결정을 분리해야 할 때

## 필요한 입력

- 이번 세션에서 읽거나 만든 산출물
- `memory/current-state.md`
- `memory/decisions.md`
- `memory/open-questions.md`

## 작업 절차

1. 이번 세션에서 바뀐 사실만 추린다.
2. 현재 진행 상태는 `current-state.md`에 갱신한다.
3. 장기적으로 유지할 방향은 `decisions.md`에 남긴다.
4. 아직 답이 없는 항목은 `open-questions.md`로 보낸다.
5. 다음 작업자가 바로 시작할 수 있는 다음 액션을 적는다.

## 기대 산출물

- 최신 상태 메모
- 결정 로그
- 정리된 미해결 질문

## 가드레일

- 장기 문서에 작업 로그 전체를 넣지 않는다.
- 이미 결정된 내용을 다시 열린 질문으로 되돌리지 않는다.
- 메모리 문서는 항상 요약형으로 유지한다.
