## 5. 종합 인사이트 및 권장사항

### 5.1 주요 발견 5가지

1. **강사 중심 강의 구조**: 67%가 단일 화자 독백 형태. 물음표(47.9회/파일)와 "되셨어요"(14.3회/파일)가 빈번하지만, 실제 학생 응답은 거의 없어 **형식적 상호작용**에 가까움.

2. **높은 반복 표현 빈도**: "그래서"(137.5회/파일), "이제"(130.3회/파일)가 과도하게 반복됨. 이는 품질 기준 1.1(불필요한 반복 표현) 평가에서 **주요 감점 요인**이 될 수 있음.

3. **구조화된 교수법**: 학습목표 안내, 전날 복습 연계, 이해확인 질문이 일관되게 관찰됨. 카테고리 2(강의 구조) 항목들의 점수가 상대적으로 높을 것으로 예상.

4. **화자 ID 비일관성**: 날짜별로 화자 ID가 완전히 달라 동일 강사의 종단적 개선 추적이 불가능. 메타데이터와의 교차 검증이 필요.

5. **18개 항목 중 50%는 규칙 기반 자동화 가능**: 반복표현, 발화속도, 이해확인 빈도 등 9개 항목은 regex/빈도 분석으로 1차 자동 채점이 가능하며, 나머지 9개는 LLM 문맥 이해가 필수.

### 5.2 파이프라인 구현 시 고려사항

| 고려사항 | 설명 | 대응 방안 |
|---------|------|----------|
| 청킹 전략 | 30분 윈도우가 적절하나 휴식 시간(10~30분 갭) 자동 감지 필요 | 타임스탬프 갭 ≥10분 시 청크 분리 |
| 학생 상호작용 | 강사 독백 위주 데이터에서 카테고리 5 평가 신뢰도 낮음 | 이해확인 질문 빈도(5.2)에 집중, 5.4는 N/A 처리 고려 |
| STT 품질 | 구두점 부재, 동음이의어 오류 | 전처리 단계에서 문장 경계 복원 로직 추가 |
| 화자 식별 | 주화자/부화자 자동 구분 필요 | 라인 수 기반 주화자 추정 (최다 발화자 = 강사) |
| 비유/예시 탐지 | "마치", "예를 들어" 등 패턴 매칭 + 문맥 판단 | 하이브리드 접근 (규칙 1차 필터 + LLM 2차 평가) |
