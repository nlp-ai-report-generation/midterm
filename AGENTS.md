# 에이전트 작업 운영 규칙

이 저장소의 에이전트는 NLP 과제 1인 "AI 강의 분석 리포트 생성기"를 더 빠르고 일관되게 진행하기 위한 공용 작업 규칙을 따른다.

## 목적

- 강의 스크립트, 메타데이터, 강의 품질 기준을 바탕으로 강사에게 도움이 되는 분석 결과와 리포트를 만든다.
- 결과물은 평가 기준에 맞춰 서비스 완성도, 커뮤니케이션, 문제 해결력, 프레젠테이션 품질을 함께 높여야 한다.
- 분석 결과는 비개발자도 이해할 수 있어야 하며, 근거와 개선 제안이 분리되어야 한다.

## 우선 참조 순서

1. 현재 사용자 요청
2. 이 문서 `AGENTS.md`
3. `memory/current-state.md`
4. `memory/decisions.md`
5. 루트 `README.md`
6. 과제 폴더 `README.md`, `강의 품질 기준.md`, `docs/decision-points.md`

우선순위가 충돌하면 상위 문서를 따른다. 다만 상위 요청이 명시적으로 기존 결정을 뒤집는 경우, 그 변경을 메모리에 남긴다.

## 시작 전 필수 절차

1. `memory/current-state.md`를 읽고 현재 목표, 최근 변경, 다음 작업을 파악한다.
2. `memory/decisions.md`와 관련 자료를 읽고 이미 정해진 방향을 확인한다.
3. 이번 작업과 직접 연결된 데이터, 문서, 코드만 추가로 읽는다.
4. 분석 또는 제안에 들어가기 전에 사실, 해석, 가정을 분리한다.

## 작업 중 규칙

- 체크리스트 항목과 직접 연결되지 않는 분석 지표를 새로 만들면, 이유와 한계를 함께 문서화한다.
- 원본 강의 스크립트, 메타데이터, 체크리스트 원문은 덮어쓰지 않는다. 파생 산출물은 별도 파일로 만든다.
- 결과는 항상 "관찰된 사실", "해석", "개선 제안"을 구분해서 쓴다.
- 점수나 평가 문구는 근거 없이 단정하지 않는다.
- STT 품질 한계, 화자 구분 부재, 문맥 손실 가능성은 필요할 때 명시한다.
- 임시 메모를 장기 문서에 그대로 복붙하지 않는다. 요약한 뒤 구조화해서 남긴다.

## 분석 및 리포트 근거 규칙

- 모든 핵심 주장은 아래 중 하나 이상에 근거해야 한다.
  - 강의 스크립트의 실제 표현
  - 강의 품질 기준 문서의 항목
  - 강의 메타데이터
  - 명시적으로 적은 휴리스틱
- 근거가 약한 경우 확정 표현 대신 가능성 표현을 쓴다.
- 개선 제안은 가능한 한 구체적인 행동 단위로 제시한다.
- 리포트는 강사의 개선 행동으로 이어질 수 있어야 한다.

## 데이터 취급 규칙

- 이 저장소에는 과제 수행용 원본 데이터가 포함될 수 있다.
- 원본 데이터는 저장소 내부 제한 자산으로 취급한다.
- 에이전트는 외부 공유, 외부 업로드, 공개 재배포를 기본 행동으로 제안하지 않는다.
- 새로 생성하는 문서에는 원본 전체를 재수록하지 말고 필요한 최소 인용 또는 요약만 남긴다.
- 민감 여부가 애매하면 원문 대신 요약과 참조 경로를 우선 사용한다.

## 커뮤니케이션 규칙

- 기본 문서는 한국어 우선으로 작성한다.
- 비개발자도 이해할 수 있는 표현을 우선한다.
- 기술 설명이 필요하면 먼저 사용자 가치와 결과를 설명한 뒤 구현 세부를 붙인다.
- 평가자, 강사, 팀원이 빠르게 훑어볼 수 있게 짧은 섹션과 명확한 제목을 사용한다.

## 종료 전 필수 절차

1. 변경 결과를 짧게 검토하고 중복 또는 모순이 없는지 확인한다.
2. 이번 작업으로 바뀐 현재 상태를 `memory/current-state.md`에 반영한다.
3. 장기적으로 유지할 결정은 `memory/decisions.md`에 남긴다.
4. 아직 미해결이면 `memory/open-questions.md`로 보낸다.
5. 다음 에이전트가 바로 이어받을 수 있게 남은 작업을 분명히 적는다.
