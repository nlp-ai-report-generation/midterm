# AI 강의 분석 리포트 — 기획서

> LangGraph 기반 에이전틱 강의 평가 파이프라인 + 운영 대시보드
>
> 멋쟁이사자처럼 AXP 인턴 1-2조 · 2026.03 ~ 2026.04

<br/>

## 왜 만들었나요

"오늘 강의 어땠나요?"

이 질문에 보통은 수강생 설문으로 답해요. 그런데 설문은 주관적이고, 매번 같은 기준으로 측정하기 어렵고, 강사에게 구체적으로 뭘 바꾸라고 말해주지 못해요.

| 문제 | 지금은 이래요 |
|------|------------|
| 시간이 오래 걸려요 | 강의 하나를 평가하는 데 수 시간이 걸려요 |
| 기준이 달라요 | 평가자마다 같은 강의도 다르게 점수를 매겨요 |
| 전부 다루기 어려워요 | 강의 수가 늘어나면 빠지는 강의가 생겨요 |

이 프로젝트는 **강의 녹취록을 AI가 직접 읽고 18개 항목으로 채점**해요. 사람이 체크하던 걸 LLM이 대신하는 거예요.

<br/>

## 이 시스템이 하는 일

한 문장으로 요약하면:

> 강의 녹취록이 들어오면 → 30분 단위로 쪼개고 → 5개 카테고리 평가자가 동시에 읽고 점수를 매기고 → 가중 평균을 내고 → 리포트를 만들어요.

```
강의 녹취록 (.txt)
    ↓
전처리 ─── 30분 윈도우 청킹 + 화자 식별
    ↓
병렬 평가 ─ 5개 카테고리 × GPT-4o-mini (동시 실행)
    ↓
집계 ───── 가중 평균 (HIGH 3 · MEDIUM 2 · LOW 1)
    ↓
보정 ───── ICC 기반 편향 교정
    ↓
리포트 ─── 강점 · 개선점 · 제안 자동 생성
```

### 핵심 설계 원칙

- **하네스 기반이에요.** 평가 기준이 코드가 아니라 마크다운 파일이에요. 개발자가 아니어도 수정할 수 있어요.
- **그래프가 알아서 만들어져요.** 하네스 파일을 추가하면 평가 노드가 자동으로 생겨요.
- **실험을 재현할 수 있어요.** 모든 실험은 config + results + metrics가 함께 저장돼요.

<br/>

## 뭘 평가하나요

5개 카테고리, 총 18개 항목으로 강의를 채점해요. 각 항목은 1~5점이에요.

| 카테고리 | 항목 수 | 주요 평가 내용 |
|----------|--------|--------------|
| 언어 표현 품질 | 3 | 반복 표현, 발화 완결성, 언어 일관성 |
| 강의 도입 및 구조 | 5 | 학습 목표, 복습 연계, 설명 순서, 마무리 |
| 개념 설명 명확성 | 4 | 개념 정의, 비유/예시, 발화 속도 |
| 예시 및 실습 연계 | 2 | 예시 적절성, 이론→실습 연결 |
| 수강생 상호작용 | 4 | 이해 확인, 참여 유도, 질문 응답 |

항목마다 가중치(HIGH / MEDIUM / LOW)가 달라요. "이해 확인 질문을 하는지"(HIGH)는 "마무리 요약이 있는지"(LOW)보다 더 중요하게 반영돼요.

## 실험으로 검증했어요

### 실험 1. 같은 강의를 3번 채점하면 같은 점수가 나올까요?

GPT-4o-mini로 15개 강의를 각 3회씩 반복 평가했어요.

| 지표 | 결과 | 판정 |
|------|------|------|
| ICC (급내상관계수) | 0.877 | ✅ Good |
| Cohen's Kappa | 0.883 | ✅ Almost Perfect |
| Krippendorff's Alpha | 0.873 | ✅ Reliable |
| SSI (점수안정성지수) | 0.974 | ✅ Very Stable |

→ 높은 수준의 평가 일관성을 확인했어요.

### 실험 2. 30분으로 자르는 것과 15분으로 자르는 것, 차이가 있을까요?

| 통계량 | 결과 |
|--------|------|
| 30분 평균 | 3.245 |
| 15분 평균 | 3.033 |
| t(14) | 4.421 |
| p-value | 0.0006 (p < 0.001) |
| Cohen's d | 1.142 (큰 효과) |

→ 청크 크기가 점수에 유의미한 영향을 줘요. 비교할 때는 반드시 같은 청크 크기를 써야 해요.
