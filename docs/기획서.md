# AI 강의 분석 리포트 — 프로젝트 기획서

> **LangGraph 기반 에이전틱 강의 평가 파이프라인 + 운영 대시보드**
>
> 멋쟁이사자처럼 AXP 인턴 1-2조 · 2026.03 ~ 2026.04

---

## 1. 프로젝트 배경

멋쟁이사자처럼 AXP에서는 매 학기 수십 개의 부트캠프 강의가 동시에 진행됩니다. 강의 품질을 관리하려면 평가가 필요한데, 기존 방식에는 세 가지 문제가 있습니다.

| 문제 | 설명 |
|------|------|
| **시간** | 강의 하나를 수동 평가하는 데 수 시간이 소요됩니다 |
| **일관성** | 평가자마다 기준이 달라 같은 강의도 점수가 들쭉날쭉합니다 |
| **확장성** | 강의 수가 늘어나면 전부 다루기 어렵습니다 |

이 프로젝트는 **강의 녹취록(STT)을 AI가 직접 읽고 18개 항목으로 자동 채점**하여, 일관되고 확장 가능한 강의 품질 평가 시스템을 구축합니다.

---

## 2. 목표

### 핵심 목표
- STT 트랜스크립트를 입력하면 **자동으로 품질 리포트가 생성**되는 End-to-End 파이프라인
- 평가 결과를 **시각적으로 확인**할 수 있는 운영 대시보드
- AI 평가의 **신뢰성을 통계적으로 검증**하는 실험 프레임워크

### 성공 지표
- ICC (급내상관계수) ≥ 0.75 — 반복 평가 일관성
- Cohen's Kappa ≥ 0.80 — 평가 결과 일치도
- 15개 강의 × 3개 모델 교차 평가 완료

---

## 3. 시스템 아키텍처

```
강의 녹취록 (.txt)
    ↓
[전처리] 30분 윈도우 청킹 + 화자 식별
    ↓
[병렬 평가] 5개 카테고리 × GPT-4o-mini (fan-out)
    ↓
[집계] 가중 평균 (fan-in)
    ↓
[보정] ICC 기반 편향 교정
    ↓
[리포트] 강점 · 개선점 · 제안 자동 생성
```

### 핵심 설계 원칙
- **하네스 기반**: 평가 기준이 코드가 아니라 마크다운 파일 — 개발자가 아니어도 수정 가능
- **동적 그래프**: 하네스 파일을 추가하면 평가 노드가 자동 생성
- **실험 재현성**: 모든 실험은 config + results + metrics가 함께 저장

## 4. 평가 체계

| 카테고리 | 항목 수 | 핵심(HIGH) | 주요 평가 내용 |
|----------|--------|-----------|--------------|
| 1. 언어 표현 품질 | 3 | 1 | 반복 표현, 발화 완결성, 언어 일관성 |
| 2. 강의 도입 및 구조 | 5 | 2 | 학습 목표, 복습 연계, 설명 순서, 마무리 |
| 3. 개념 설명 명확성 | 4 | 2 | 개념 정의, 비유/예시, 발화 속도 |
| 4. 예시 및 실습 연계 | 2 | 2 | 예시 적절성, 이론→실습 연결 |
| 5. 수강생 상호작용 | 4 | 3 | 이해 확인, 참여 유도, 질문 응답 |

**총 18개 항목**, 각 항목 1~5점 채점, 가중치(HIGH/MEDIUM/LOW) 적용 가중 평균

## 8. 실험 프레임워크

### 실험 1. 평가 일관성 (Test-Retest)
- GPT-4o-mini로 15개 강의 × 3회 반복 평가
- ICC 0.877, Cohen's κ 0.883, Krippendorff's α 0.873
- **결론**: 높은 수준의 평가 일관성 확인

### 실험 2. 청크 크기 민감도
- 30분 vs 15분 청크 비교 (대응표본 t-test)
- t(14) = 4.421, p < 0.001, Cohen's d = 1.142
- **결론**: 청크 크기가 점수에 유의미한 영향 → 비교 시 동일 설정 필수
