# 도메인 모델

## 핵심 엔티티

### 강의 세션

- 하루 단위 강의 기록
- 하나의 스크립트 파일과 메타데이터 행에 대응할 수 있다

### 강의 스크립트

- STT 기반 텍스트 원문
- 분석의 1차 증거 자료
- 문장 분리 오류, 화자 혼합, 오인식 가능성이 있다

### 강의 메타데이터

- 날짜, 과목명, 강사명, 시간 등 강의 배경 정보
- 비교 분석, 그룹화, 리포트 표시용 기준 정보로 사용한다

### 강의 품질 항목

- 체크리스트의 카테고리와 세부 항목
- 분석과 점수화의 기준 프레임

### 분석 결과

- 특정 강의 세션 또는 여러 세션에 대한 관찰 결과
- 근거 문장, 해석, 점수, 개선 제안으로 구성될 수 있다

### 점수

- 품질 항목별 정량 또는 준정량 평가값
- 체크리스트 기준 또는 별도 휴리스틱 설명과 함께 제시해야 한다

### 리포트

- 강사 또는 평가자가 읽는 최종 산출물
- 요약, 관찰, 근거, 개선 제안, 필요 시 시각화로 구성한다

## 관계

- 하나의 강의 세션은 하나의 강의 스크립트와 연결된다.
- 하나의 강의 세션은 하나의 메타데이터 행과 연결된다.
- 하나의 분석 결과는 하나 이상의 품질 항목을 다룰 수 있다.
- 하나의 리포트는 하나 이상의 분석 결과를 묶는다.

## 용어 통일

- "강의 품질 기준", "체크리스트", "평가 항목"은 문맥에 따라 구분해서 쓴다.
- "근거"는 원문 인용 또는 관찰된 데이터 조각을 뜻한다.
- "해석"은 근거를 기반으로 한 분석자의 판단이다.
- "개선 제안"은 해석을 바탕으로 한 행동 권고다.
