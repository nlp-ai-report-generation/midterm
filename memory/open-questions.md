# 열린 질문

## 평가 품질

### 하네스 프롬프트 튜닝이 필요한가

- 상태: 스모크 테스트 후 확인 예정
- 영향: 점수의 변별력과 근거 품질에 직접 영향

### 보정(Calibrator) 노드가 실제로 도움이 되는가

- 상태: A/B 실험으로 확인 예정 (calibrator on vs off)
- 영향: 파이프라인 복잡도와 비용에 영향

### 최적 청킹 윈도우 크기는 얼마인가

- 상태: 실험으로 확인 예정 (30분 vs 45분 vs 60분)
- 영향: 평가 정확도와 API 비용의 트레이드오프

## 산출물 및 데모

### 중간평가 시연에서 무엇을 보여줄 것인가

- 상태: 팀 논의 필요
- 영향: 2주차까지의 개발 우선순위가 달라진다

### 대시보드 UI를 어느 수준까지 구현할 것인가

- 상태: 미결정
- 영향: Streamlit 스캐폴드가 이미 존재. 범위 결정 필요

### 전체 15개 강의 실제 평가 결과를 언제 프론트 정적 데이터로 반영할 것인가

- 상태: 스모크 테스트 1건만 반영 완료, 전체 배치 실행은 미수행
- 영향: 현재 데모는 실제 평가 결과 1건만 보여주며, 전체 코호트 비교 화면은 배치 평가 완료 전까지 제한됨

## 운영

### 강사별 비교 분석까지 포함할 것인가

- 상태: 미결정 (현재 데이터는 단일 강사)
- 영향: 현재 데이터로는 불가. 향후 다중 강사 데이터 확보 시 확장
