# 열린 질문

## 평가 품질

### 하네스 프롬프트 튜닝이 필요한가

- 상태: 스모크 테스트 후 확인 예정
- 영향: 점수의 변별력과 근거 품질에 직접 영향

### 보정(Calibrator) 노드가 실제로 도움이 되는가

- 상태: A/B 실험으로 확인 예정 (calibrator on vs off)
- 영향: 파이프라인 복잡도와 비용에 영향

### 최적 청킹 윈도우 크기는 얼마인가

- 상태: 실험으로 확인 예정 (30분 vs 45분 vs 60분)
- 영향: 평가 정확도와 API 비용의 트레이드오프

## 산출물 및 데모

### 중간평가 시연에서 무엇을 보여줄 것인가

- 상태: 팀 논의 필요
- 영향: 2주차까지의 개발 우선순위가 달라진다

### 대시보드 UI를 어느 수준까지 구현할 것인가

- 상태: 미결정
- 영향: Streamlit 스캐폴드가 이미 존재. 범위 결정 필요

### 전체 15개 강의 실제 평가 결과를 언제 프론트 정적 데이터로 반영할 것인가

- 상태: 스모크 테스트 1건만 반영 완료, 전체 배치 실행은 미수행
- 영향: 현재 데모는 실제 평가 결과 1건만 보여주며, 전체 코호트 비교 화면은 배치 평가 완료 전까지 제한됨

### TRIBE v2 코랩 노트북을 실제 공식 추론 코드와 어느 수준까지 연결할 것인가

- 상태: 실제 `TribeModel.from_pretrained` / `get_events_dataframe(text_path=...)` / `model.predict()` 경로로 갱신했지만 실데이터 1회 실행은 아직 필요
- 영향: 실제 시뮬레이션 신뢰도와 발표 시 설명 가능성에 직접 영향

### TRIBE raw output의 좌/우 반구 분할 규약이 코랩 후처리와 정확히 일치하는가

- 상태: 실데이터 검증 필요
- 영향: 3D 히트맵의 정점 색상 매핑 정확도에 직접 영향

### audio-only 최적화 노트북이 기존 fallback 대비 실제로 얼마나 빨라졌는가

- 상태: 최적화는 적용 완료, 코랩에서 A100/H100 기준 실측 필요
- 영향: 파일럿 3강의 실행 계획과 발표 시 데모 가능 범위에 직접 영향

### `2026-02-09`, `2026-02-24`도 `2026-02-02`와 같은 ROI 화면 계약으로 확장할 수 있는가

- 상태: `2026-02-02`만 완주 raw 결과 기준으로 프론트 반영 완료, 나머지 날짜는 partial 또는 raw 없음
- 영향: 시뮬레이션 기능을 단일 강의 데모에서 파일럿 3강의 비교 기능으로 확장할 수 있는지에 직접 영향

### live 화면용 진짜 timestep cortical frame을 코랩 산출물에 어떻게 저장할 것인가

- 상태: 현재 프론트 live 화면은 transcript line timestamp + 세그먼트 평균 반응 + 원문 기반 heuristic remap fallback으로 동작함
- 영향: 사용자가 라인 단위로 보는 brain 변화가 실제 timestep raw를 반영하는지, 혹은 세그먼트 평균을 반복 노출하는지에 따라 신뢰도와 설명 방식이 달라짐

## 운영

### 제출용 `midterm/main`을 현재 작업 브랜치 기준으로 어떻게 합칠 것인가

- 상태: 현재 커밋 `8dc070d8`는 `midterm/feature/tribe-roi-20260202` 브랜치까지는 푸시 완료. 하지만 `midterm/main`은 unrelated history라 일반 merge에서 대규모 add/add 충돌이 발생해 force 없이 바로 반영하지 못함.
- 영향: 제출용 리포지토리의 기본 브랜치를 최신 작업 상태로 맞추려면, 어떤 트리를 기준으로 보존할지 먼저 결정해야 한다.

### 강사별 비교 분석까지 포함할 것인가

- 상태: 미결정 (현재 데이터는 단일 강사)
- 영향: 현재 데이터로는 불가. 향후 다중 강사 데이터 확보 시 확장
