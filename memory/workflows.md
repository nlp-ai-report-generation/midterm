# 표준 작업 흐름

## 1. 분석 항목 설계

1. `강의 품질 기준.md`에서 대상 항목을 고른다.
2. 해당 항목이 규칙 기반, LLM 기반, 하이브리드 중 어디에 맞는지 정한다.
3. 입력 데이터와 증거 단위를 정한다.
4. 한계와 오판 가능성을 함께 기록한다.

## 2. 단일 강의 분석

1. 대상 스크립트와 메타데이터를 확인한다.
2. 분석 항목별 근거 문장을 찾는다.
3. 관찰, 해석, 개선 제안 순서로 정리한다.
4. 필요하면 점수 또는 등급을 부여하고 이유를 붙인다.

## 3. 리포트 초안 작성

1. 보고 대상과 목적을 정한다.
2. 핵심 요약을 먼저 쓴다.
3. 주요 관찰과 근거를 배치한다.
4. 실행 가능한 개선 제안을 붙인다.
5. 비개발자 관점에서 문장을 다듬는다.

## 4. 의사결정 정리

1. 장기적으로 유지할 선택인지 확인한다.
2. 결정, 이유, 결과를 `memory/decisions.md`에 남긴다.
3. 아직 미확정이면 `memory/open-questions.md`에 넣는다.

## 5. 세션 종료

1. `memory/current-state.md`의 최근 완료와 다음 작업을 갱신한다.
2. 새로 생긴 리스크나 블로커를 추가한다.
3. 다음 사람이 바로 시작할 수 있을 정도로 문장을 짧게 정리한다.
